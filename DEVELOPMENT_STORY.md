# 以太坊巨鲸监控：我写了个脚本盯着有钱人的钱包，这合法吗

> 区块链上所有交易都是公开的。所以我写了个 Python 脚本，专门盯着那些一次转 100 个 ETH 以上的地址。每次有大鱼动了，我的 Obsidian 笔记本就多一条记录。像是在写一本"有钱人今天又干了什么"的日记。

---

## 起因：看了一篇推文，热血上头

事情是这样的。

某天刷推特（对不起，X），看到有人发了一条：

> "某巨鲸刚从币安提出 3000 ETH，上次这个地址这么干的时候，ETH 三天后涨了 15%。"

底下几百条评论，全在分析这个地址的历史操作。

我当时的反应：**等等，这些数据不是公开的吗？为什么我要等别人告诉我？**

以太坊区块链上每一笔交易都是透明的。你不需要任何人的许可，就可以看到任何地址在任何时间转了多少钱给谁。

这不是黑客行为，这是区块链的**设计初衷**。

所以我决定：**自己盯。**

---

## 什么是"巨鲸"

在加密货币世界里，"巨鲸"（Whale）是指持有大量加密货币的地址。

当一个地址一次性转出 100 个 ETH（大概 30 万美金），这通常意味着**有人要搞事**：

- 从交易所提到冷钱包 → 长期持有信号（看好后市）
- 从冷钱包转到交易所 → 可能要卖（看空后市）
- 大额转账到未知地址 → 可能是 OTC 交易、机构操作、或者新项目部署

普通散户的 0.1 ETH 转账不会影响市场。但当一个地址一次转 3000 ETH 的时候，**整个市场都在看。**

我想做的就是：**比大多数人更早看到。**

## 技术方案：简单到令人发指

### 核心逻辑

```
每5秒 → 查最新区块 → 遍历所有交易 → 找到 ≥ 100 ETH 的 → 记录
```

就这么简单。没有机器学习，没有预测模型，没有花里胡哨的东西。

纯粹的**数据监控**。

### 数据来源

**Etherscan API V2** — 以太坊区块链的 Google。免费 tier 够用，每秒 5 次请求。

查两样东西：
1. 最新区块号
2. 这个区块里的所有交易

### ETH 价格

用 **Yahoo Finance** 实时获取 ETH/USD 价格。

为什么不用币安 API？因为 Yahoo Finance 不需要 API Key，而且我只需要一个大概的美元估值，不是在做高频交易。

### 地址标签

提前建了一个地址库，标记了已知的大户：

| 地址 | 标签 |
|------|------|
| 0x4dbd... | 🔶 币安-14 |
| 0x28c6... | 🔶 币安-热钱包 |
| 0xdfd5... | 💎 OKX-热钱包 |
| 0x3027... | 🐳 Upbit-巨鲸 |
| 0x4645... | 🏦 Jump Trading |

当监控到的交易来自或发往这些地址时，自动打标签。

"未知地址"更有意思——它们可能是新的机构钱包、项目方、或者某个不想被认出来的大佬。

## Obsidian 集成：每条交易变成一篇笔记

这是我最喜欢的部分。

每次监控到巨鲸交易，脚本会自动在 `Whale_Archives/` 目录下生成一个 Markdown 文件：

```markdown
---
amount: 134.26 ETH
usd_value: $392,172
from: 0x1234...（🔶 币安-热钱包）
to: 0xabcd...（未知地址）
timestamp: 2026-01-22 14:33:21
---

# 🐳 巨鲸交易记录

**金额**: 134.26 ETH ($392,172)
**方向**: 币安 → 未知地址
**链接**: [Etherscan](https://etherscan.io/tx/0x...)
```

为什么存到 Obsidian？因为我的整个知识管理系统都在 Obsidian 里。巨鲸交易数据和我的市场分析笔记可以**双向链接**——看到某天 ETH 暴涨，点开关联笔记，发现三天前有巨鲸从交易所大量提币。

**事后诸葛亮？是的。但至少有数据可以复盘。**

## 几件值得记录的事

### 区块链的透明性是一把双刃剑

所有交易都是公开的，这意味着：
- 你可以监控任何人的钱包（合法）
- 任何人也可以监控你的钱包（也合法）

"匿名"在区块链上是一个**幻觉**——你只是用了一个随机地址而不是名字。但只要你在某个交易所做过 KYC，那个交易所知道你是谁，也就意味着这个地址和你的真实身份之间只隔着一个数据库查询。

### v2 说明有 v1

文件名叫 `whale_v2.py`，意味着有一个 v1 被我扔了。

v1 大概是在试错阶段的产物——API 用法不对、数据解析有 bug、阈值设得太低导致通知轰炸。具体细节已经不记得了，但 v2 明显更干净。

### 这个项目让我理解了"链上分析"

以前觉得"链上分析"是很玄乎的东西。做了这个项目之后发现：**核心逻辑极其简单——就是遍历区块里的每一笔交易，然后按条件筛选。**

复杂的部分不在技术，在于**你知不知道哪些条件是有意义的**。知道一个地址是币安热钱包，和不知道，分析出来的结论完全不同。
