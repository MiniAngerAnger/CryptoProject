# PROJECT_WORKFLOW.md

## CryptoProject 标准开发流程（含 GitHub 研究）

### 0) 目标确认（先写清）
- 本轮要解决什么问题（1 句话）
- 验收标准是什么（可验证）
- 本轮是否涉及外部动作（消息推送/公开发布）

---

### 1) GitHub 研究流程（新增必做）
每轮开发前先做一次轻量研究，避免闭门造车。

1. 关键词检索（`gh search repos`）
2. 选 3 个参考仓库（一个大而稳、一个轻量、一个与你目标最接近）
3. 提取可复用点：
   - 架构分层
   - 数据模型
   - 稳定性处理（重试/降级/日志）
4. 输出“借鉴边界”：
   - 只借鉴思路，不抄实现
   - 不引入重框架重写现有代码
5. 形成一段研究结论（5-10 行）再进入开发

---

### 2) 方案设计（小步）
- 拆成 3~6 个可交付小步骤
- 每步明确：改哪些文件、风险点、回滚方式

---

### 3) 实施开发
- 先加可运行最小版本（MVP）
- 外部 API 必须可降级
- 数据库改动统一走 `src/storage/db.py`

---

### 4) 自测与验收
最低验收：
- `python3 -m py_compile ...` 通过
- 主线程启动无崩溃
- 关键日志字段齐全
- dashboard/接口可看到新能力

---

### 5) 仓库收尾（防漏提）
- `git status --short` 检查
- `git ls-files` 检查关键文件是否纳管
- `.gitignore` 只忽略运行产物/隐私文件

---

### 6) 文档同步（最少 3 处）
- `PROJECT_META.md`
- `DEVELOPMENT_STORY.md`
- `LEARNING_NOTES.md`

---
